# Repository Notes for Agents

Global conventions (commits, branching, PRs, markdown/shell/JSON style,
security scanners) are inherited from `~/.config/opencode/AGENTS.md`. Only
repo-specific facts live here.

## What this repo is

Notes-only repository ("Claude Courses Notes"). No application code, no build,
no tests, no package manifest. `README.md` is a two-line stub. Do not invent
build/test workflows - there are none to run locally.

## CI is the only enforcement

All quality gates run in CI via **MegaLinter** (`.mega-linter.yml`). There is
no `pre-commit`, no `Makefile`, no task runner. To reproduce a check locally,
invoke the individual tool (`rumdl`, `shellcheck`, `shfmt`, `lychee`,
`jsonlint`, `actionlint`, `checkov`, `trivy`, `devskim`, `kingfisher`) with
the args from `.mega-linter.yml`.

Non-obvious MegaLinter choices already configured - do not "fix" them:

- Markdown: `rumdl` (not markdownlint); config in `.rumdl.toml`.
- Link checking: `lychee` (not markdown-link-check); config in `lychee.toml`.
- IaC security: `checkov` + `trivy` only (terrascan is disabled).
- Spell check (`cspell`) is disabled - do not enable.
- `CHANGELOG.md` is excluded from all linters and is auto-generated by
  `release-please` (`.github/workflows/release-please.yml`). Never hand-edit
  it.
- `.devcontainer/devcontainer.json` is excluded from `jsonlint`.

## Workflow file changes

`.github/workflows/` also gets scanned by `zizmor` (via MegaLinter) in
addition to `actionlint`. Keep `permissions:` minimal and pin actions to full
SHAs - CI will flag both.

## Releases

Version bumps and `CHANGELOG.md` are managed by `release-please`. Commit
messages therefore must be strict Conventional Commits (already enforced by
`commit-check.yml` and `semantic-pull-request.yml`) - the release notes
depend on them.
